# コンテキスト：getContext OR検索実装

## 1. 目的・背景

### 実装目的

- `getContext` 関数のコンテキスト文字列を形態素解析で複数キーワードに展開してOR検索する機能の実装
- 単一キーワードの完全一致検索から、文脈的に関連するキーワード群での横断検索への拡張
- ヒット件数爆発を防ぐスコアリング（複数キーワードヒット数）と件数上限の導入

### ビジネス要件

- ユーザが入力したコンテキスト文字列（自然言語・複合語・パス文字列）から有効なキーワードを抽出できること
- 複数キーワードのいずれかにヒットしたファイルを重複なく返すこと
- キーワード抽出後に有効なキーワードが0件の場合、元の `context` をそのまま使用すること
- 結果件数を上位20件に制限し、ヒットキーワード数が多いファイルを上位に表示すること

---

## 2. 機能要件

### 2.1 形態素解析によるキーワード抽出

- **使用ライブラリ**: `kuromoji`（`kuromoji` + `@types/kuromoji`）
- **抽出対象品詞**: 名詞・動詞（基本形）・英数字
- **除外品詞**: 助詞・助動詞・接続詞・形容詞・形容動詞・記号
- **動詞の正規化**: `token.basic_form` を使用（活用形を基本形に統一）
- **フォールバック**: 有効トークンが0件の場合、元の `context` 文字列をそのまま1件のキーワードとして使用

### 2.2 並列OR検索

- 有効なキーワードそれぞれで `searchContent` を並列実行（`Promise.all`）
- `repository_id` + `path` をキーに重複排除してマージ
- 各ファイルについて「ヒットしたキーワード数」をスコアとしてカウント
- スコア降順でソートし、上位20件のみを返す

### 2.3 ターゲット解決ロジック

- 既存の `repository_id` 指定・CWD前方一致・全リポジトリフォールバックのロジックは変更しない

---

## 3. 非機能要件

### 3.1 パフォーマンス

- `searchContent` は `Promise.all` で並列実行する（逐次実行は禁止）
- 結果上限: 20件（スコア降順）

### 3.2 影響範囲最小化

- 変更は `src/context-provider.ts` のみ（`file-searcher.ts`・`types.ts` は変更しない）

## 9. 実装方針

### 9.1 開発手法

- **TDD（テスト駆動開発）**: Red-Green-Refactorサイクル厳守
- **テストファースト**: 実装前に必ずテストを書く

### 9.2 コーディング規約

- **TypeScript strict mode 準拠**
- **命名規則**:
  - 関数・変数: `camelCase`
  - 型・インターフェース: `PascalCase`
- キーワード抽出ロジックは `src/keyword-extractor.ts` に `extractKeywords(context: string): Promise<string[]>` としてエクスポートする
- `context-provider.ts` は `extractKeywords` を import して使用する

### 9.3 実装ポイント

**`src/keyword-extractor.ts`**
- kuromoji のビルダー初期化は Promise でラップして非同期化する
- 抽出対象品詞: `token.pos === "名詞"` または `token.pos === "動詞"` または `token.pos === "英数"` または `token.pos_detail_1 === "固有名詞"`
- 動詞は `token.basic_form` を使用（`"*"` の場合は `token.surface_form` にフォールバック）
- 重複トークンは除外する

**`src/context-provider.ts`**
- `extractKeywords` の戻り値が空配列の場合、`[context]` をフォールバックとして使用
- 重複排除キー: `` `${result.repository_id}::${result.path}` ``
- スコアリング: `Map<string, { result: ContextResult; score: number }>` でキー別にスコアを加算
- 並列実行: `await Promise.all(keywords.map(kw => searchContent({ keyword: kw, configs: targets })))`
- 件数上限: スコア降順ソート後 `.slice(0, 20)`
