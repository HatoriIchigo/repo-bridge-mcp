# コンテキスト：F-001 リポジトリ登録機能実装

## 1. 目的・背景

### 実装目的

- `.repo-bridge/<repo-id>.json` を読み込み、登録リポジトリ一覧を管理するモジュールの実装
- `list_repositories` MCPツールへの組み込みにより、クライアントが登録済みリポジトリを取得できるようにする

### ビジネス要件

- `enabled: true` のリポジトリのみを一覧に含める
- 設定ファイルの読み込みエラー時もサーバがクラッシュせず、エラーを適切に処理する

---

## 2. 機能要件

### 2.1 設定ファイル読み込み

- **対象ディレクトリ**: MCPサーバ起動時のカレントディレクトリ直下の `.repo-bridge/`
- **対象ファイル**: `.repo-bridge/` 直下の `*.json` ファイル（サブディレクトリは対象外）
- **読み込み処理**:
  - 各JSONファイルを `RepositoryConfig` として解析する
  - `enabled: false` のエントリは結果から除外する
  - JSONパースエラーが発生したファイルはスキップし、残りのファイルの処理を継続する
  - `.repo-bridge/` ディレクトリが存在しない場合は空の配列を返す

### 2.2 list_repositories MCPツール

- **ツール名**: `list_repositories`
- **引数**: なし
- **返却内容**: `enabled: true` の `Repository` オブジェクト一覧
- **レスポンス形式**: MCPツールのcontentとして `Repository[]` のJSON文字列を返す

| 型安全性 | TypeScript strict: true を維持する |

## 4. 技術要件

- **言語**: TypeScript 5.x（strict: true）
- **ランタイム**: Node.js 24.x
- **モジュール形式**: ESM（`"type": "module"`、import拡張子に `.js` を使用）
- **MCPフレームワーク**: @modelcontextprotocol/sdk ^1.0.0
- **テストフレームワーク**: Jest（`node --experimental-vm-modules node_modules/.bin/jest`）
  - ESM環境のためjest.config に `transform: {}` と `extensionsToTreatAsEsm` の設定が必要
  - `@jest/globals` からimportして使用する
- **Node.js組み込みモジュール**: `fs/promises`（ファイル読み込み）、`path`（パス操作）

## 5. データモデル

### RepositoryConfig（`src/types.ts`）

| フィールド | 型 | 必須 | 説明 |
|-----------|-----|------|------|
| `id` | `string` | 必須 | リポジトリ識別子 |
| `name` | `string` | 必須 | リポジトリ表示名 |
| `path` | `string` | 必須 | 絶対パス |
| `enabled` | `boolean` | 必須 | 有効フラグ |
| `exclude_patterns` | `string[]` | 必須 | 除外パターン一覧 |

### Repository（`src/types.ts`）

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `id` | `string` | リポジトリ識別子 |
| `name` | `string` | リポジトリ表示名 |
| `path` | `string` | 絶対パス |

> `loadRepositories()` は `enabled: true` のエントリのみを返すため、`Repository` に `enabled` フィールドは持たない。

### 7.2 テストケース

#### 正常系

1. JSONファイルが1件・enabled: true の場合に1件返す

   **given**:
   - `.repo-bridge/my-repo.json` に以下の内容が存在する
     ```json
     {"id":"my-repo","name":"My Repository","path":"/repos/my-repo","enabled":true,"exclude_patterns":["node_modules"]}
     ```

   **when**:
   - `loadRepositories()` を実行する

   **then**:
   - `[{ id: "my-repo", name: "My Repository", path: "/repos/my-repo" }]` が返る

2. JSONファイルが複数件・全て enabled: true の場合に全件返す

   **given**:
   - `.repo-bridge/repo-a.json`: `{"id":"repo-a","name":"Repo A","path":"/repos/a","enabled":true,"exclude_patterns":[]}`
   - `.repo-bridge/repo-b.json`: `{"id":"repo-b","name":"Repo B","path":"/repos/b","enabled":true,"exclude_patterns":[]}`

   **when**:
   - `loadRepositories()` を実行する

   **then**:
   - 長さ2の配列が返り、`id` に `"repo-a"` と `"repo-b"` が含まれる

3. enabled: false のエントリを除外する

   **given**:
   - `.repo-bridge/enabled.json`: `{"id":"enabled","name":"Enabled","path":"/repos/enabled","enabled":true,"exclude_patterns":[]}`
   - `.repo-bridge/disabled.json`: `{"id":"disabled","name":"Disabled","path":"/repos/disabled","enabled":false,"exclude_patterns":[]}`

   **when**:
   - `loadRepositories()` を実行する

   **then**:
   - 長さ1の配列が返り、`id` が `"enabled"` のみ含まれる

#### 異常系

4. `.repo-bridge/` ディレクトリが存在しない場合に空配列を返す

   **given**:
   - `.repo-bridge/` ディレクトリが存在しない

   **when**:
   - `loadRepositories()` を実行する

   **then**:
   - `[]` が返る（例外をスローしない）

5. JSONパースエラーのファイルをスキップして残りを返す

   **given**:
   - `.repo-bridge/broken.json`: `{invalid json}`
   - `.repo-bridge/valid.json`: `{"id":"valid","name":"Valid","path":"/repos/valid","enabled":true,"exclude_patterns":[]}`

   **when**:
   - `loadRepositories()` を実行する

   **then**:
   - 長さ1の配列が返り、`id` が `"valid"` のみ含まれる（例外をスローしない）

6. 必須フィールドが欠落したJSONをスキップする

   **given**:
   - `.repo-bridge/incomplete.json`: `{"id":"incomplete","name":"Incomplete"}` （`path`・`enabled` が欠落）
   - `.repo-bridge/complete.json`: `{"id":"complete","name":"Complete","path":"/repos/complete","enabled":true,"exclude_patterns":[]}`

   **when**:
   - `loadRepositories()` を実行する

   **then**:
   - 長さ1の配列が返り、`id` が `"complete"` のみ含まれる

#### 境界値

7. `.repo-bridge/` が空ディレクトリ（JSONファイルが0件）の場合に空配列を返す

   **given**:
   - `.repo-bridge/` ディレクトリが存在するが `.json` ファイルが0件

   **when**:
   - `loadRepositories()` を実行する

   **then**:
   - `[]` が返る

8. サブディレクトリ内のJSONを対象外とする

   **given**:
   - `.repo-bridge/sub/repo.json`: `{"id":"sub-repo","name":"Sub","path":"/repos/sub","enabled":true,"exclude_patterns":[]}`

   **when**:
   - `loadRepositories()` を実行する

   **then**:
   - `[]` が返る（サブディレクトリのファイルを読み込まない）

9. `.json` 以外のファイルを無視する

   **given**:
   - `.repo-bridge/config.yaml`: `id: yaml-repo`
   - `.repo-bridge/notes.txt`: `some text`

   **when**:
   - `loadRepositories()` を実行する

   **then**:
   - `[]` が返る
