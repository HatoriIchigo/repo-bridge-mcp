# コンテキスト：プロジェクト初期セットアップ

## 1. 目的・背景

### 実装目的

- `repo-bridge-mcp` プロジェクトの開発環境を構築し、MCPサーバ実装を開始できる状態にする
- TypeScript + Node.js + @modelcontextprotocol/sdk によるMCPサーバのスケルトンを作成する

### ビジネス要件

- 参照: `docs/design.md`（設計ドキュメント）

---

## 2. 機能要件

### 2.1 セットアップ対象

| No | 対象 | 内容 |
|----|------|------|
| 1 | `package.json` | プロジェクト名 `repo-bridge-mcp`、エントリポイント `dist/index.js`、ビルド・開発・テストコマンド定義 |
| 2 | `tsconfig.json` | `target: ES2022`、`module: Node16`、`outDir: ./dist`、`rootDir: ./src`、`strict: true` |
| 3 | 依存パッケージ | `@modelcontextprotocol/sdk`（本番依存）、`typescript`・`@types/node`（開発依存） |
| 4 | `src/index.ts` | stdioトランスポートを使ったMCPサーバのエントリポイントスケルトン。ツールハンドラは未実装のスタブで可 |
| 5 | ビルドコマンド | `npm run build` で `dist/` へのトランスパイル |
| 6 | 開発コマンド | `npm run dev` で `ts-node` または `tsx` 経由の起動 |
| 7 | テストコマンド | `npm test` でテストランナー起動（テストファイル未作成時はスクリプト定義のみ） |

### 2.2 MCPサーバスケルトンの要件

- stdioトランスポート（`StdioServerTransport`）を使用
- サーバ名: `repo-bridge-mcp`
- 以下の5ツールをスタブとして登録（ハンドラはNotImplementedエラーを返す）

| ツール名 | 説明 |
|---------|------|
| `list_repositories` | 登録済みリポジトリの一覧取得 |
| `search_files` | ファイル名・パターンによる横断検索 |
| `read_file` | 指定ファイルの内容取得 |
| `search_content` | ファイル内容のキーワード検索 |
| `get_context` | 作業コンテキストに応じた関連ファイル取得 |

---

## 3. 非機能要件

- **セキュリティ**: `.repo-bridge/` 配下の設定ファイルに含まれるリポジトリパス外へのアクセスは禁止（初期セットアップ時点では設定読み込み処理は対象外）
- **TypeScript strict モード**: `strict: true` を必ず有効化

## 6. テスト要件

### 6.1 開発手法

- スケルトン段階のため、TDDは適用しない
- スケルトン完成後の各ツール実装時にTDD（Red-Green-Refactorサイクル）を適用する

### 6.2 動作確認テストケース（手動確認）

1. ビルド成功確認
    **given**:
        - `package.json`・`tsconfig.json`・`src/index.ts` が作成済み
        - 依存パッケージがインストール済み（`node_modules/` 存在）
    **when**:
        - `npm run build` を実行
    **then**:
        - `dist/index.js` が生成される
        - エラー・警告が0件

2. MCPサーバ起動確認
    **given**:
        - `dist/index.js` が生成済み
    **when**:
        - `node dist/index.js` を実行
    **then**:
        - プロセスが起動し、stdioで待機状態になる（クラッシュしない）

3. TypeScript型チェック通過確認
    **given**:
        - `src/index.ts` が作成済み
    **when**:
        - `npx tsc --noEmit` を実行
    **then**:
        - 型エラーが0件

## 7. 制約事項・前提条件

### スコープ内
- `package.json`・`tsconfig.json`・`src/index.ts` の新規作成
- 依存パッケージのインストール（`npm install`）
- ビルド・開発・テストコマンドの定義

### スコープ外
- 各MCPツール（`list_repositories` 等）の実際の処理実装
- テストコードの作成（スケルトン段階では不要）
- `.repo-bridge/` 設定ファイルの読み込み処理
- `src/test/` 配下のテストファイル作成

### 前提条件
- Node.js 20.x 以上がインストール済み
- npm が使用可能

---

## 8. 実装方針

### 8.1 コーディング規約

- **命名規則**:
  - 変数・関数: `camelCase`
  - クラス・型: `PascalCase`
  - 定数: `UPPER_SNAKE_CASE`
- **型ヒント**: 全ての関数引数・戻り値に型を明示（`any` 禁止）
- **strict モード**: `tsconfig.json` の `strict: true` を維持

### 8.2 使用エージェント

- なし（初期セットアップは単純なファイル作成タスクのため）
