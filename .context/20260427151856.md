# コンテキスト：設計書（docs/design.md）作成

## 1. 目的・背景

### プロジェクト背景
- プロジェクト名: `repo-bridge-mcp`
- 複数リポジトリに分散したコード・ドキュメントを、現在の作業コンテキストに応じて横断的に参照するMCPサーバ
- 必要な情報のみを動的に取得することで、ノイズを抑えつつ安全にプロジェクト全体の理解を補助することを目的とする

---

## 2. 設計書に含めるべき内容

`docs/design.md` は `.claude/rules/design.md` の記載ガイドに従い、以下のセクションで構成する。

### 2.1 概要（Overview）
- 何を作るのか（MCPサーバとしての役割）
- 誰のためか（AIアシスタント + 開発者）
- どんな価値を提供するか（リポジトリ横断参照によるコンテキスト補完）

### 2.2 アーキテクチャ設計
- mermaidを使用したアーキテクチャ概要図
- コンポーネント構成：MCPサーバ本体・リポジトリアダプタ・設定管理
- データの流れ：クライアント（Claude Code等）→ MCPサーバ → 各リポジトリ

### 2.3 機能一覧
以下の機能IDで整理する：

| 機能ID | 機能名 | 優先度 | 概要 |
|--------|--------|--------|------|
| F-001 | リポジトリ登録 | 高 | 参照対象リポジトリをプロジェクト配下の`.repo-bridge/<repo-id>.json`で管理（手動作成） |
| F-002 | ファイル検索 | 高 | 登録リポジトリ横断でファイルを検索・取得 |
| F-003 | コンテキスト取得 | 高 | 作業コンテキストに応じた関連ファイルの動的取得 |
| F-004 | ドキュメント参照 | 中 | docs配下のMarkdownファイルの参照 |
| F-005 | コード参照 | 中 | ソースコードファイルの参照 |

### 2.4 MCP ツール設計（API設計に相当）
MCPサーバはHTTPではなくstdioトランスポートを使用するため、MCPツール一覧で定義する：

| ツール名 | 概要 |
|---------|------|
| `list_repositories` | 登録済みリポジトリの一覧取得 |
| `search_files` | ファイル名・パターンによる横断検索 |
| `read_file` | 指定ファイルの内容取得 |
| `search_content` | ファイル内容のキーワード検索 |
| `get_context` | 作業コンテキストに応じた関連ファイル取得 |

### 2.5 データモデル
- `RepositoryConfig`：`.repo-bridge/<repo-id>.json` の1ファイル = 1リポジトリ設定（id, name, path, enabled, exclude_patterns）
- `Repository`：実行時のリポジトリ表現（id, name, path, enabled）— `RepositoryConfig`から生成
- `FileEntry`：ファイルエントリ（path, repository_id, type）
- `SearchResult`：検索結果（file_entry, snippet, relevance_score）

### 2.6 非機能要件
- **パフォーマンス**: ファイル検索レスポンス 1000ms以内（95パーセンタイル）
- **セキュリティ**: 登録リポジトリ外へのパストラバーサル禁止
- **設定**: `.repo-bridge/<repo-id>.json`（リポジトリごと1ファイル）による除外パターン管理

### 2.7 制約・前提条件
- トランスポート: stdio（ローカル実行のみ、リモート接続は対象外）
- 対応リポジトリ: ローカルファイルシステム上のgitリポジトリ
- スコープ外: コード実行・変更・コミット操作

## 5. 設計方針・根拠

| 決定事項 | 選定内容 | 根拠 |
|---------|---------|------|
| MCPフレームワーク | @modelcontextprotocol/sdk | 公式SDK、TypeScriptサポートが最も充実 |
| トランスポート | stdio | Claude Codeとのローカル統合が最もシンプル |
| 設定管理 | `.repo-bridge/<repo-id>.json`（リポジトリごと個別ファイル） | リポジトリ単位で独立管理でき、追加・削除が1ファイル操作で完結する |
