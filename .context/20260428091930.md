# コンテキスト：get_context 自律呼び出し促進

## 1. 目的・背景

### 実装目的

- ClaudeなどのAIアシスタントが、ユーザーから明示的に指示されなくても `get_context` を自律的に呼び出すよう促す
- MCPツールの `description` とサーバレベルの `instructions` を改善し、呼び出し判断の手がかりを提供する

### ビジネス要件

- ユーザーが「MCPを使って」と指示しなくても、作業開始前に `get_context` が自動的に呼び出されること
- 既存の動作（CWD判定・repository_id指定）は変更しないこと

### 背景・前提

- F-003（`get_context` 実装）が完了済み
- `@modelcontextprotocol/sdk` 1.29.0 では `McpServer` コンストラクタの第1引数に `instructions?: string` フィールドが存在する
- 参照: `src/index.ts`（変更対象）

---

## 2. 機能要件

### 2.1 変更箇所

| ファイル | 変更箇所 | 変更内容 |
|---------|---------|---------|
| `src/index.ts` | `McpServer` コンストラクタ | `instructions` フィールドを追加 |
| `src/index.ts` | `get_context` ツールの第2引数（description） | より詳細な説明文に変更 |

### 2.2 McpServer instructions の追加

`McpServer` コンストラクタの第1引数オブジェクトに `instructions` を追加する。

**変更前:**

```typescript
const server = new McpServer({
  name: "repo-bridge-mcp",
  version: "0.1.0",
});
```

**変更後:**

```typescript
const server = new McpServer({
  name: "repo-bridge-mcp",
  version: "0.1.0",
  instructions:
    "作業を開始する前に必ず get_context を呼び出し、関連ファイルを取得すること。" +
    "コードの実装・デバッグ・レビュー・質問への回答など、いかなる作業においても最初のステップとして実行すること。",
});
```

### 2.3 get_context ツール description の変更

**変更前:**

```
"作業コンテキストに応じた関連ファイル取得"
```

**変更後:**

```
"【作業開始前に必ず呼び出す】実装・デバッグ・レビュー・質問への回答など、いかなる作業においても最初に呼び出し、関連ファイルとスニペットを取得する。contextには作業内容・キーワードを渡す。"
```

---

## 3. 非機能要件

- 既存の全テストがパスし続けること
- `npm run build` でコンパイルエラーが発生しないこと
- `src/index.ts` 以外のファイルは変更しないこと

---

## 4. 技術要件

- **言語**: TypeScript 5.x（`strict: true`）
- **変更対象**: `src/index.ts` のみ
- **テスト**: 変更なし（`src/index.ts` はテスト対象外）
