# コンテキスト：F-003 コンテキスト取得機能実装

## 1. 目的・背景

### 実装目的

- MCPツール `get_context` の実装（現状はスタブ）
- 作業ディレクトリ（CWD）に基づいて検索対象リポジトリを自動特定
- キーワードでファイル内容を横断検索し、マッチ箇所のスニペットを返す

### ビジネス要件

- 開発者が作業中のリポジトリを自動検出し、関連ファイルを提示すること
- `repository_id` を明示した場合、CWD判定を上書きして指定リポジトリのみを対象とすること
- 登録リポジトリ外のパスへのアクセスを一切禁止すること

### 背景・前提

- F-001（リポジトリ登録）・F-002（ファイル検索）が実装済みであることを前提とする
- 参照: `docs/design.md`（F-003: 優先度「高」）、`.context/20260427153000.md`（F-001実装コンテキスト）、`.context/20260427160000.md`（F-002実装コンテキスト）
- `src/index.ts` の `get_context` ツールはスタブ（`throw new Error("Not implemented")`）として登録済み

## 2. 機能要件

### 2.1 get_context ツール

**入力パラメータ:**

| パラメータ | 型 | 必須 | 説明 |
|-----------|-----|------|------|
| `context` | string | 必須 | 検索キーワード・作業内容の説明 |
| `repository_id` | string | 省略可 | 明示指定する場合のリポジトリID |

**処理ロジック（優先順）:**

1. `repository_id` が指定されている場合 → 指定リポジトリのみを対象（以降のCWD判定をスキップ）
2. `repository_id` が未指定の場合:
   a. `process.cwd()` でCWDを取得
   b. 登録リポジトリの `path` に対して `cwd.startsWith(repoPath + "/") || cwd === repoPath` で判定
   c. 複数マッチする場合は最長一致のリポジトリを採用
   d. 1件もマッチしない場合は全登録リポジトリを対象
3. 対象リポジトリのファイルを `context`（キーワード）で内容検索（`file-searcher.ts` の `searchContent` を使用）
4. マッチしたファイルのパスとスニペット（ヒット行の前後3行）を返す

**返却型:**

```
{ repository_id: string; path: string; snippet: string }[]
```

### 2.2 スニペット生成仕様

- ヒット行の前後3行（合計最大7行）を含める
- ファイル先頭付近でヒットした場合（前3行が存在しない）: 存在する行のみ含める（最小1行）
- ファイル末尾付近でヒットした場合（後3行が存在しない）: 存在する行のみ含める（最小1行）
- 複数ヒットがある場合: 各ヒット行につきそれぞれスニペットを生成し、同一ファイルの結果を1エントリとして結合する

### 2.3 エラーハンドリング

- `context` が空文字（`""`）の場合: `Error: context is required` をthrow
- `repository_id` が存在しないIDの場合: `Error: Repository not found: {id}` をthrow
- リポジトリパスが存在しないディレクトリの場合: そのリポジトリをスキップし、他リポジトリの結果を返す
- パストラバーサルが検出された場合: `Error: Path traversal is not allowed` をthrow

## 3. 非機能要件

### 3.1 セキュリティ

- 登録リポジトリ外へのパストラバーサル禁止
- ファイルアクセス前に `resolve()` で正規化し、リポジトリルートの前方一致（`fullPath.startsWith(repoRoot + "/") || fullPath === repoRoot`）で検証

### 3.2 パフォーマンス

- ファイル検索レスポンス 1000ms以内（95パーセンタイル）

## 6. 既存コードの依存関係

### 使用する既存関数・追加する関数

| 関数 | ファイル | 種別 | シグネチャ・役割 |
|------|---------|------|----------------|
| `loadRepositories` | `src/repository-manager.ts` | 既存 | `(baseDir?: string): Promise<RepositoryConfig[]>` |
| `readFileContent` | `src/file-searcher.ts` | 既存 | `(options: { repository_id, path, configs }): Promise<string>` |
| `searchContent` | `src/file-searcher.ts` | 新規追加 | `(options: { keyword: string; configs: RepositoryConfig[] }): Promise<ContextResult[]>` — 全ファイルを走査しキーワードが含まれる行を特定、スニペットを生成して返す |

> `searchFiles`（globパターンによるファイル名検索）は本機能では使用しない。ファイル内容検索は `searchContent` で行う。

### 追加する型（src/types.ts）

`ContextResult` 型を追加する。フィールド: `repository_id`（string）、`path`（string）、`snippet`（string）。

### src/index.ts の変更箇所

- `get_context` ツールの引数に `repository_id: z.string().optional()` を追加
- ハンドラ内の `throw new Error("Not implemented")` を `context-provider.ts` の実装呼び出しに差し替え

#### 正常系

**TC-N-001: CWDが登録リポジトリのパスと完全一致**

- **given**:
  - 登録リポジトリ: `{ id: "repo-a", path: "/home/user/repo-a", enabled: true, exclude_patterns: [] }`
  - CWD: `/home/user/repo-a`
  - `context`: `"authentication"`
  - `/home/user/repo-a/src/auth.ts` の内容に `"authentication"` が含まれる
- **when**: `getContext({ context: "authentication" })` を呼び出す
- **then**: `[{ repository_id: "repo-a", path: "src/auth.ts", snippet: <ヒット行前後3行> }]` を返す

**TC-N-002: CWDが登録リポジトリのサブディレクトリ**

- **given**:
  - 登録リポジトリ: `{ id: "repo-a", path: "/home/user/repo-a", enabled: true, exclude_patterns: [] }`
  - CWD: `/home/user/repo-a/src/components`
  - `context`: `"Button"`
  - `/home/user/repo-a/src/components/Button.ts` の内容に `"Button"` が含まれる
- **when**: `getContext({ context: "Button" })` を呼び出す
- **then**: `[{ repository_id: "repo-a", path: "src/components/Button.ts", snippet: <ヒット行前後3行> }]` を返す

**TC-N-003: CWDが登録リポジトリ外 → 全リポジトリ検索**

- **given**:
  - 登録リポジトリ: `repo-a (path: "/home/user/repo-a")`, `repo-b (path: "/home/user/repo-b")`
  - CWD: `/home/user/other-dir`
  - `context`: `"config"`
  - `repo-a` の `config.ts` に `"config"` が含まれる
  - `repo-b` の `settings.ts` に `"config"` が含まれる
- **when**: `getContext({ context: "config" })` を呼び出す
- **then**: `repo-a` と `repo-b` の両方の結果を含む配列を返す

**TC-N-004: repository_id 指定あり（CWDはリポジトリ外） → 指定リポジトリのみ**

- **given**:
  - 登録リポジトリ: `repo-a`, `repo-b`
  - CWD: `/home/user/other-dir`
  - `repository_id`: `"repo-b"`
  - `context`: `"config"`
  - `repo-a` の `config.ts` に `"config"` が含まれる
  - `repo-b` の `settings.ts` に `"config"` が含まれる
- **when**: `getContext({ context: "config", repository_id: "repo-b" })` を呼び出す
- **then**: `repo-b` の結果のみを含む配列を返す（`repo-a` の結果は含まない）

**TC-N-005: repository_id 指定あり（CWDは別リポジトリ内） → 指定リポジトリが優先**

- **given**:
  - 登録リポジトリ: `repo-a (path: "/home/user/repo-a")`, `repo-b (path: "/home/user/repo-b")`
  - CWD: `/home/user/repo-a`
  - `repository_id`: `"repo-b"`
  - `context`: `"config"`
  - `repo-b` の `settings.ts` に `"config"` が含まれる
- **when**: `getContext({ context: "config", repository_id: "repo-b" })` を呼び出す
- **then**: `repo-b` の結果のみを含む配列を返す（`repo-a` の結果は含まない）

**TC-N-006: マッチするファイルなし → 空配列返却**

- **given**:
  - 登録リポジトリ: `repo-a`
  - CWD: `/home/user/repo-a`
  - `context`: `"zzz_no_match_keyword_xyz"`
  - リポジトリ内のいずれのファイルにもキーワードが含まれない
- **when**: `getContext({ context: "zzz_no_match_keyword_xyz" })` を呼び出す
- **then**: `[]` を返す

**TC-N-007: 複数リポジトリがCWDに前方一致 → 最長一致採用**

- **given**:
  - 登録リポジトリ: `repo-parent (path: "/home/user/repo")`, `repo-child (path: "/home/user/repo/child")`
  - CWD: `/home/user/repo/child/src`
  - `context`: `"feature"`
  - `repo-child` の `feature.ts` に `"feature"` が含まれる
  - `repo-parent` の `main.ts` に `"feature"` が含まれる
- **when**: `getContext({ context: "feature" })` を呼び出す
- **then**: `repo-child` の結果のみを含む配列を返す（`repo-parent` の結果は含まない）

#### 境界値テスト

**TC-B-001: スニペット — ファイル先頭行（1行目）にヒット**

- **given**:
  - ファイル内容: 10行
  - 1行目に `"TARGET_KEYWORD"` が含まれる
  - `context`: `"TARGET_KEYWORD"`

**TC-B-002: スニペット — ファイル末尾行（最終行）にヒット**

- **given**:
  - ファイル内容: 10行
  - 10行目に `"TARGET_KEYWORD"` が含まれる
  - `context`: `"TARGET_KEYWORD"`
- **when**: `getContext({ context: "TARGET_KEYWORD" })` を呼び出す
- **then**: スニペットが7〜10行目の内容（後3行が存在しないため前3行のみ）を含む

**TC-B-003: スニペット — ファイル中央行（4行目）にヒット**

- **given**:
  - ファイル内容: 10行
  - 4行目に `"TARGET_KEYWORD"` が含まれる
  - `context`: `"TARGET_KEYWORD"`
- **when**: `getContext({ context: "TARGET_KEYWORD" })` を呼び出す
- **then**: スニペットが1〜7行目の内容（前3行: 1〜3行目、ヒット: 4行目、後3行: 5〜7行目）を含む
