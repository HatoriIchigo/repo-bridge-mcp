# コンテキスト：settings.json path変数展開の実装

## 1. 目的・背景

### 実装目的
- `settings.json` の `path` フィールドに `${HOME}` および `${CUR}` という変数を書けるようにする
- 展開は `loadRepositories()` / `loadSettings()` 内で完結させ、他ファイルへの変更を一切行わない
- `RepositoryConfig.path` として返す時点で展開済み絶対パスになっている状態を保証する

### ビジネス要件
- ユーザが `settings.json` をポータブルに管理できること（ホームディレクトリやカレントディレクトリへの依存を環境変数で抽象化）
- 既存の `settings.json`（変数なし・絶対パス直書き）は変更なしで引き続き動作すること

---

## 2. 機能要件

### 2.1 サポートする変数

| 変数名 | 展開先 | 取得方法 |
|--------|--------|----------|
| `${HOME}` | ホームディレクトリ | `process.env.HOME` |
| `${CUR}` | カレントディレクトリ | `process.cwd()` |

### 2.2 展開仕様
- `path` 文字列中に現れる全ての `${HOME}` / `${CUR}` をそれぞれの値で置換する
- 先頭以外に変数が含まれる場合も展開する（例: `/base/${HOME}/sub`）
- 展開後の値は `RepositoryConfig.path` として格納し、返却する
- `${HOME}` が参照する `process.env.HOME` が `undefined` または空文字 (`""`) の場合はエラーをスローする
- `${CUR}` が参照する `process.cwd()` が空文字 (`""`) の場合はエラーをスローする

### 2.3 エラー仕様
- エラーメッセージ: `Variable ${HOME} is not defined` / `Variable ${CUR} is not defined`
- エラー型: `Error`
- エラーはスローし、呼び出し元に伝播させる（握りつぶさない）

### 2.4 展開タイミング
- `loadSettings()` の中で `isRepositoryConfig()` 検証後、`path` を展開してから `Settings` オブジェクトに格納する
- または `loadRepositories()` で `filter` 後に展開する（どちらか実装上自然な方を選択する）
- 展開処理は専用のプライベート関数 `expandPathVariables(path: string): string` として切り出す

---

## 3. 非機能要件

### 3.1 影響範囲
- 変更対象ファイル: `src/repository-manager.ts`（1ファイルのみ）
- `src/types.ts` / `src/file-searcher.ts` / `src/index.ts` は変更しない

### 3.2 後方互換性
- 変数を含まない `path` は展開処理を通過しても値が変わらないこと

## 6. テスト要件（TDD）

### 6.1 開発手法
- **TDD（テスト駆動開発）**: Red-Green-Refactorサイクル厳守
- **テストファースト**: 実装前に必ずテストを書く

### 6.2 テストケース

#### 正常系テスト

**TC10: `${HOME}` を含む path が process.env.HOME の値に展開されること**
- **given**:
  - `process.env.HOME` = `/home/testuser`
  - settings.json の path = `${HOME}/repos/my-repo`
  - enabled = `true`
- **when**:
  - `loadRepositories(baseDir)` を実行
- **then**:
  - 返却された `RepositoryConfig.path` = `/home/testuser/repos/my-repo`

**TC11: `${CUR}` を含む path が process.cwd() の値に展開されること**
- **given**:
  - `process.cwd()` = `/workspace`（モック）
  - settings.json の path = `${CUR}/repos/my-repo`
  - enabled = `true`
- **when**:
  - `loadRepositories(baseDir)` を実行
- **then**:
  - 返却された `RepositoryConfig.path` = `/workspace/repos/my-repo`

**TC12: `${HOME}` と `${CUR}` を含まない path は変更されないこと**
- **given**:
  - settings.json の path = `/absolute/path/to/repo`
  - enabled = `true`
- **when**:
  - `loadRepositories(baseDir)` を実行
- **then**:
  - 返却された `RepositoryConfig.path` = `/absolute/path/to/repo`（変更なし）

**TC13: path 内に `${HOME}` と `${CUR}` が両方含まれる場合、両方展開されること**
- **given**:
  - `process.env.HOME` = `/home/testuser`
  - `process.cwd()` = `/workspace`（モック）
  - settings.json の path = `${HOME}/base/${CUR}/sub`
  - enabled = `true`
- **when**:
  - `loadRepositories(baseDir)` を実行
- **then**:
  - 返却された `RepositoryConfig.path` = `/home/testuser/base//workspace/sub`

**TC14: path 内に `${HOME}` が複数回現れる場合、全て展開されること**
- **given**:
  - `process.env.HOME` = `/home/testuser`
  - settings.json の path = `${HOME}/${HOME}`
  - enabled = `true`
- **when**:
  - `loadRepositories(baseDir)` を実行
- **then**:
  - 返却された `RepositoryConfig.path` = `/home/testuser//home/testuser`

#### 異常系テスト

**TC15: `${HOME}` を含む path で process.env.HOME が undefined の場合、エラーをスローすること**
- **given**:
  - `process.env.HOME` = `undefined`（環境変数を削除）
  - settings.json の path = `${HOME}/repos/my-repo`
  - enabled = `true`
- **when**:
  - `loadRepositories(baseDir)` を実行
- **then**:
  - `Error: Variable ${HOME} is not defined` がスローされる

**TC16: `${HOME}` を含む path で process.env.HOME が空文字の場合、エラーをスローすること**
- **given**:
  - `process.env.HOME` = `""`
  - settings.json の path = `${HOME}/repos/my-repo`
  - enabled = `true`
- **when**:
  - `loadRepositories(baseDir)` を実行
- **then**:
  - `Error: Variable ${HOME} is not defined` がスローされる

**TC17: `${CUR}` を含む path で process.cwd() が空文字を返す場合、エラーをスローすること**
- **given**:
  - `process.cwd` が `""` を返すようにモック
  - settings.json の path = `${CUR}/repos/my-repo`
  - enabled = `true`
- **when**:
  - `loadRepositories(baseDir)` を実行
- **then**:
  - `Error: Variable ${CUR} is not defined` がスローされる

#### 境界値テスト

**TC18: path が `${HOME}` のみの場合、ホームディレクトリそのものに展開されること**
- **given**:
  - `process.env.HOME` = `/home/testuser`
  - settings.json の path = `${HOME}`
  - enabled = `true`
- **when**:
  - `loadRepositories(baseDir)` を実行
- **then**:
  - 返却された `RepositoryConfig.path` = `/home/testuser`

**TC19: path が空文字の場合、展開処理が行われず空文字のまま返ること**
- **given**:
  - settings.json の path = `""`
  - enabled = `true`
- **when**:
  - `loadRepositories(baseDir)` を実行
- **then**:
  - 返却された `RepositoryConfig.path` = `""`（変数展開は行われない）

---

## 7. 制約事項・前提条件

- `${HOME}` と `${CUR}` 以外の変数（例: `${USER}`, `${PATH}`）は展開しない（無視する）
- 変数の部分展開（`$HOME` 形式、`%HOME%` 形式）はサポートしない
- `src/types.ts` の `RepositoryConfig.path` の型定義（`string`）は変更しない
- `src/file-searcher.ts` は変更しない

---

## 8. コーディング規約

- 既存コードのスタイルに準拠（`function` 宣言、`import type` 使用）
- 新規関数 `expandPathVariables` は `repository-manager.ts` 内のプライベート関数として定義
- JSDocコメントを付与する
- `any` 型の使用禁止
