# コンテキスト：F-002 ファイル検索機能実装

## 1. 目的・背景

### 実装目的

- F-002 ファイル検索機能の実装（`search_files` / `read_file` MCPツール）
- 登録リポジトリ横断でのglobパターンによるファイル検索
- 指定ファイルの内容取得（パストラバーサル禁止）

### ビジネス要件

- 参照: `docs/design.md`（機能一覧・MCPツール設計・データモデル・非機能要件）
- F-001（リポジトリ登録）が実装済みであることを前提とする

---

## 2. 機能要件

### 2.1 search_files ツール

- **入力**:
  - `pattern`（必須、string）: ファイル名のglobパターン（例: `*.ts`、`src/**/*.md`）
  - `repository_id`（省略可、string）: 対象リポジトリID。省略時は全登録リポジトリ対象
  - `configs`（必須、`RepositoryConfig[]`）: 呼び出し元から注入するリポジトリ設定一覧（疎結合のため）
- **処理**:
  - `configs` から対象リポジトリを絞り込む（`repository_id` 指定時は該当のみ。該当なしは空配列）

  - 各リポジトリのルートディレクトリを `fs/promises` で再帰的に走査し、globパターンにマッチするファイルを収集
  - `RepositoryConfig.exclude_patterns` に該当するパスを結果から除外する
  - `exclude_patterns` の照合はリポジトリルートからの相対パスに対して行う
  - globパターンマッチングは外部ライブラリを使用せず、Node.js標準APIのみで実装する
- **返却**: `FileEntry[]`（`{ repository_id: string, path: string, type: "file" | "directory" }`）

### 2.2 read_file ツール

- **入力**:
  - `repository_id`（必須、string）: リポジトリID
  - `path`（必須、string）: リポジトリルートからの相対パス
  - `configs`（必須、`RepositoryConfig[]`）: 呼び出し元から注入するリポジトリ設定一覧（疎結合のため）
- **処理**:
  - `configs` から指定IDのリポジトリを特定。該当なしの場合は `Error: Repository not found: {repository_id}` をthrow
  - `path.resolve(repoRoot, inputPath)` でフルパスを構築
  - 構築したフルパスがリポジトリルート（`repoRoot + "/"`）で始まらない場合、`Error: Path traversal is not allowed` をthrow
  - ファイル内容をUTF-8文字列として返却
- **返却**: ファイル内容文字列（string）

> `src/index.ts` では `loadRepositories()` を呼び出して取得した `RepositoryConfig[]` を `configs` として渡す。`file-searcher.ts` は `loadRepositories()` に依存しない。
>
> `loadRepositories()` の返却型は `RepositoryConfig[]` に変更する。`list_repositories` ツールのレスポンスは `index.ts` 側で `{ id, name, path }` に絞って整形する（`exclude_patterns` はAPIレスポンスに含めない）。

## 3. 非機能要件

| 項目 | 要件 |
|------|------|
| パフォーマンス | ファイル検索レスポンス 1000ms以内（95パーセンタイル） |
| セキュリティ | 登録リポジトリ外へのパストラバーサルは `Error: Path traversal is not allowed` をthrowして拒否 |
| 除外パターン | `RepositoryConfig.exclude_patterns` を使用し、リポジトリルート相対パスに対して照合 |

## 6. テスト要件（TDD）

### 6.1 開発手法

- **TDD（テスト駆動開発）**: Red-Green-Refactorサイクル厳守
- **テストファースト**: 実装前に必ずテストを書く
- **カバレッジ目標**: 100%を目指す（90%を超過しても打ち切らない）

### 6.2 テストケース（given-when-then形式）

#### 正常系

**TC-F002-001: 全リポジトリ対象のglobパターン検索**
- given:
  - リポジトリA（id: `repo-a`）に `src/main.ts`・`src/utils.ts`・`README.md` が存在する
  - リポジトリB（id: `repo-b`）に `src/index.ts`・`docs/api.md` が存在する
  - `exclude_patterns: []`（両リポジトリ）
- when: `searchFiles({ pattern: "*.ts" })` を実行
- then: `repository_id` と `path` を含む `FileEntry` が3件返る（`repo-a/src/main.ts`・`repo-a/src/utils.ts`・`repo-b/src/index.ts`）

**TC-F002-002: repository_id指定での絞り込み検索**
- given:
  - リポジトリA（id: `repo-a`）に `src/main.ts` が存在する
  - リポジトリB（id: `repo-b`）に `src/index.ts` が存在する
- when: `searchFiles({ pattern: "*.ts", repository_id: "repo-a" })` を実行
- then: `repo-a` の `src/main.ts` 1件のみが返る

**TC-F002-003: 除外パターンによるファイル除外**
- given:
  - リポジトリA（id: `repo-a`）に `src/main.ts`・`node_modules/lib/index.ts` が存在する
  - `exclude_patterns: ["node_modules"]`
- when: `searchFiles({ pattern: "*.ts" })` を実行
- then: `src/main.ts` 1件のみが返る（`node_modules/lib/index.ts` は除外される）

**TC-F002-004: read_fileによるファイル内容取得**
- given:
  - リポジトリA（id: `repo-a`）に `src/main.ts` が存在し、内容は `const x = 1;`
- when: `readFile({ repository_id: "repo-a", path: "src/main.ts" })` を実行
- then: `"const x = 1;"` が返る

**TC-F002-005: マッチするファイルが0件**
- given:
  - リポジトリA（id: `repo-a`）に `README.md` のみ存在する
  - `exclude_patterns: []`
- when: `searchFiles({ pattern: "*.ts" })` を実行
- then: 空配列が返る

#### 異常系

**TC-F002-006: 存在しないrepository_idを指定したsearch_files**
- given: 登録リポジトリに `repo-a` のみ存在する
- when: `searchFiles({ pattern: "*.ts", repository_id: "non-existent" })` を実行
- then: 空配列が返る

**TC-F002-007: 存在しないrepository_idを指定したread_file**
- given: 登録リポジトリに `repo-a` のみ存在する
- when: `readFile({ repository_id: "non-existent", path: "src/main.ts" })` を実行
- then: `Error: Repository not found: non-existent` がthrowされる

**TC-F002-008: パストラバーサル攻撃の拒否（read_file）**
- given:
  - リポジトリA（id: `repo-a`、path: `/repos/a`）が存在する
- when: `readFile({ repository_id: "repo-a", path: "../secret.txt" })` を実行
- then: `Error: Path traversal is not allowed` がthrowされる

**TC-F002-009: 存在しないファイルのread_file**
- given:
  - リポジトリA（id: `repo-a`）が存在する
- when: `readFile({ repository_id: "repo-a", path: "nonexistent.ts" })` を実行
- then: Node.jsの `ENOENT` エラー（`code: "ENOENT"`）がthrowされる

#### 境界値

**TC-F002-010: patternが空文字**
- given:
  - リポジトリA（id: `repo-a`）に `src/main.ts` が存在する
- when: `searchFiles({ pattern: "" })` を実行
- then: 空配列が返る

**TC-F002-011: リポジトリが0件登録の場合**
- given: `.repo-bridge/` ディレクトリが存在しない
- when: `searchFiles({ pattern: "*.ts" })` を実行
- then: 空配列が返る

**TC-F002-012: パストラバーサルだが同じリポジトリ内に戻るケース（read_file）**
- given:
  - リポジトリA（id: `repo-a`、path: `/repos/a`）に `src/main.ts` が存在し、内容は `const x = 1;`
- when: `readFile({ repository_id: "repo-a", path: "src/../src/main.ts" })` を実行（`path.resolve` 後 `/repos/a/src/main.ts`）
- then: `"const x = 1;"` が返る（リポジトリ内に留まるため許可）

## 7. コーディング規約

- TypeScript `strict: true`（`any` 禁止）
- 全パブリック関数にJSDoc（`/** */`形式）を付与
- 命名規則: 関数・変数は `camelCase`、型は `PascalCase`
- インポート拡張子: `.js` を使用（ESM要件）
- 非同期: `fs/promises` の `readFile`・`readdir` を使用（同期APIは禁止）
- globパターンマッチングに外部ライブラリは使用しない
