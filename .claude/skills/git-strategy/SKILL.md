## Gitブランチ命名規則

### ブランチ命名規則

- **main**: メインブランチ。商用リリース用。
- **staging**: 検証用ブランチ。
- **develop**: 開発用ブランチ。
- **feature/<機能名>**: 新機能追加に伴うブランチ
- **bugfix/<バグ名>**: バグ修正に伴うブランチ
- **hotfix/<バグ名>**: 緊急バグ修正に伴うブランチ

### ブランチ命名例

1. feature/create-login-page
    ログインページの追加

2. hotfix/crash-on-startup
    アプリ起動時にクラッシュする問題を修正


## commitメッセージ規則

### タイトル

「コミット種別：要約」でコミットタイトルを記載。

1. コミット種別
    - **fix**： バグ修正
    - **add**：新規（ファイル）機能追加
    - **update**：機能修正（バグではない）
    - **remove**：削除（ファイル）

2. 要約
    - コミットの内容を記載

3. 例
    - fix: 削除フラグが更新されない不具合の修正
    - add: ログイン機能の追加

### Subject

コミットメッセージ本文の記載内容。

1. 書く内容
    - なぜその変更をしたのか（理由）
    - 何をどう変えたのか（概要）
    - 影響範囲

2. 書かない内容
    - テストをPASSしたかどうか

### コミットメッセージ例

```
fix: ログイン時にエラーが発生する問題を修正

原因：
- セッションの有効期限チェックが漏れていた

対応：
- トークン検証ロジックを追加
- リフレッシュトークンの再取得処理を実装
```
